# Claude Code ニュースレポート 2026-04-07

## 最新ニュース・アップデート

### OpenClaw利用に追加料金が必要に（2026年4月4日）

Anthropicは、Claude Codeのサブスクリプションユーザーに対し、OpenClawなどのサードパーティツールの利用に追加料金が発生することを発表しました。今後はサブスクリプションの利用枠とは別に、従量課金（pay-as-you-go）での支払いが必要になります。

- Claude Code責任者のBoris Cherny氏は「サブスクリプションはサードパーティツールの利用パターンを想定して設計されていない」と説明
- 移行措置として、月額料金相当の一回限りのボーナスクレジット（4月17日まで）と、プリペイドバンドル最大30%割引を提供
- OpenClawから開始し、今後他のサードパーティハーネスにも段階的に適用予定

### Claude Codeソースコード流出事件（2026年3月31日〜）

Anthropicが定期的なソフトウェアアップデート中に、Claude Codeの内部ソースコード約512,000行（TypeScript）を誤って公開。同社は「人的ミス」と説明しています。

### 2026年4月のClaude Code技術アップデート

**新機能**
- **フックイベントの拡張**: PreToolUseフックに「defer」パーミッション判定を追加。ヘッドレスセッションでツール呼び出し時に一時停止し、`-p --resume`で再開可能に
- **フリッカーフリー描画**: `CLAUDE_CODE_NO_FLICKER=1`環境変数でちらつきのないalt-screen描画に対応
- **PermissionDeniedフック**: オートモード分類器が拒否した後に発火する新フック
- **名前付きサブエージェント**: @メンションのタイプアヘッド候補にサブエージェント名を表示
- **MCP接続の非ブロッキング化**: `MCP_CONNECTION_NONBLOCKING=true`で`-p`モード時のMCP接続待ちをスキップ可能
- **/powerupコマンド**: Claude Code機能をアニメーションデモで学べるインタラクティブレッスン（4月1日追加）
- **/release-notes**: 対話型バージョンピッカーに進化
- **Bedrock設定ウィザード**: ログイン画面から「サードパーティプロバイダ」経由で利用可能に

**パフォーマンス改善**
- Writeツールの差分計算が大きなファイル（タブ/&/$を含む）で60%高速化
- WASMのyoga-layoutをピュアTypeScript実装に置き換え、スクロール性能改善
- 長時間セッションの安定性向上

**バグ修正**
- レート制限ダイアログが無限ループしてセッションクラッシュする問題を修正
- Windows・PowerShell環境での権限問題を修正

### 2026年3月の主要アップデート振り返り

- **音声モード**: `/voice`コマンドで有効化。Push-to-Talk方式で日本語含む20言語対応、追加コストなし
- **マルチエージェント対応**: 並列エージェントがそれぞれ独自のコンテキストウィンドウを持ち、共有タスクリストで連携
- **スケジュールタスク**: 定期的なソフトウェア作業をオートモード・リモート制御・クラウド自動修正と組み合わせて実行
- **デスクトップ操作**: Mac上でAPIなしに直接デスクトップやアプリを操作可能に
- **メッセージングアプリ統合**: Telegram、Discord、iMessageと連携し、通知受信・アクション承認が可能に
- **Claude Sonnet 4.6**: エージェント検索性能の向上とトークン消費の削減を実現

### 実践的なTips・ベストプラクティス

1. **コンテキスト管理が最重要**: コンテキストウィンドウが埋まるとLLMの性能が低下。`/clear`でタスク間をリセットし、70%使用時点で積極的に`/compact`でコンパクト化
2. **計画を先に立てる**: Claudeが各判断で80%正しいとしても、20の判断を含む機能では成功率はわずか1%。`Shift+Tab`で計画モードに切り替え、シニアアーキテクトのように考えさせる
3. **CLAUDE.mdファイルの活用**: ビルドコマンド、コードスタイル、アーキテクチャの方針を50〜100行で記述。`.gitignore`と同じくらい重要。各行について「これを削除したらClaudeがミスするか？」と問い、不要なら削除
4. **タスクの委譲**: テスト生成、ボイラープレート、マイグレーション、ドキュメント、フォーマットはサブエージェントに全面委譲
5. **並列エージェントの活用**: 複数のClaudeセッションが独自のコンテキストウィンドウを持ち、共有タスクリストで連携。1つがチームリーダーとして機能
6. **プリントモード（`-p`フラグ）**: CIパイプラインやスクリプトに組み込み可能
