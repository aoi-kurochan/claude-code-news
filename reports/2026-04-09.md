# Claude Code ニュースレポート 2026-04-09

## 最新ニュース・アップデート

### Claude Mythos Preview 発表（4月7日）
Anthropicは「Claude Mythos Preview」を「Project Glasswing」を通じて11の企業・組織に限定提供開始。サイバーセキュリティの脆弱性発見と修正に特化したモデルで、OpenBSD・FFmpeg・Linuxを含む多数のゼロデイ脆弱性を発見する能力を持つ。

### OpenClawなどサードパーティハーネスの課金変更（4月4日）
Anthropicは2026年4月4日正午（太平洋時間）より、Claude Codeのサブスクリプション枠をOpenClawなどのサードパーティハーネスに適用できないと発表。外部ツール経由での利用には別途従量課金プランが必要となった。

### Claude Code最新バージョン（v2.1.90）
最新バージョンでは以下の機能が追加・改善された：
- Bedrockセットアップウィザード（ログイン画面から利用可能）
- コスト表示の強化とインタラクティブなリリースノートピッカー
- リモートコントロールのデフォルト設定改善
- 大規模ファイル書き込みの高速化

### Claude Buddy（エイプリルフール機能）
Anthropicが2026年4月1日のエイプリルフール企画として「Claude Buddy」を公開。ターミナル上で動作するタマゴッチ風のアシスタントで、開発セッションに反応して動作する。完成度の高さから多くの開発者が正式機能として存続を希望している。

### ソースコードリーク（3月31日）
バージョン2.1.88にJavaScript Source Mapファイルが誤って含まれ、約512,000行の非難読化TypeScriptコード（約1,900ファイル）が公開される事態が発生。AI開発のアーキテクチャ構造が詳細に明らかになった。

### 3月の主要アップデート
- `--bare`オプション（スクリプト向けコール）と`--channels`権限リレー（スマートフォンへの承認プロンプト転送）の追加
- インタラクティブなチャート・図表・ビジュアライゼーション・モバイルアプリの生成が可能に
- クラウド自動修正機能：WebおよびモバイルセッションがPRのフォロー、CI失敗の修正、コメント対応を自動実行
- `/powerup`コマンド追加：機能をアニメーションで学べるインタラクティブレッスン

### 4月の追加アップデート詳細（releasebot.io より）
- **Amazon Bedrock（Mantle経由）サポート**: `CLAUDE_CODE_USE_MANTLE=1` 設定で有効化
- **デフォルト努力レベルをhighに変更**: APIキー・Bedrock/Vertex/Foundry・Team・Enterpriseユーザー対象
- **MCP サイズ上限を500Kに大幅拡大**: より大きなコンテキストを外部ツールで扱えるように
- **チラつきのないalt-screen描画（flicker-free）**: 長時間セッションの表示安定性が向上
- **名前付きサブエージェント**: @メンションで特定エージェントを指定できるように
- **ヘッドレスモード権限のdeferral対応**: 自動化ワークフローでの権限管理が改善
- **`/cost`コマンドの情報充実**: セッションのコスト詳細をより詳しく表示
- **Bedrockセットアップウィザード**: ログイン画面から簡単にセットアップ可能

## 活用事例・Tips

### 非エンジニアによる業務自動化
- 請求書確認作業が4.5時間→35分に短縮（87%削減）
- アソビュー社CPOが「地味だけど毎日使うAI自動化を11個作った」事例を公開
- 2026年はエンジニア以外も全員が使うべきツールとの認識が広がる

### 開発効率化
- バグ修正：エラー原因を約15分で特定、従来の半日作業を1.5時間で完了
- 大規模プロジェクトで前提知識ゼロからバグ修正を効率的に実施（TVer Tech Blog）
- 企業全社導入後、数日で数万行のコードを生成した事例も

### コンテキスト管理の重要Tips
- `/clear`でコンテキストリセット、`/compact`で長いセッションを圧縮
- 複数のWorktreeを使い10〜15セッションを同時並行実行（チームの最大の生産性Tips）
- CLAUDE.mdには「Claudeが間違えたことを追記する」運用が定着

### 実用的なTips（2026年4月版）
1. **メモリ機能**：`#`から始めるだけでauto-memoryに保存
2. **やり直し機能**：`/rewind`（またはESCを2回）で変更前に復帰
3. **外部エディタ利用**：`$EDITOR`環境変数でシンタックスハイライト付き編集
4. **モデル切り替え**：Opus・Sonnet・HaikuをタスクにI応じて使い分け
5. **サブエージェント活用**：`.claude/agents/`にエージェント定義を置き専門化

### シニアエンジニアの活用戦略
- コーディング前に必ず計画フェーズを設け、Plan Modeでレビュー・承認後に実装開始
- Skillsファイルを使い、CLAUDE.mdを軽量に保ちながら必要時のみ知識をロード
- Git Worktreeで複数Claudeセッションが同一プロジェクト内で競合なく作業
