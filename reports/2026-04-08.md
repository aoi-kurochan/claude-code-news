# Claude Code ニュースレポート 2026-04-08

## 最新ニュース・アップデート

### サードパーティツールの課金変更（2026年4月4日）
Anthropicは2026年4月4日正午（太平洋時間）より、Claude Codeサブスクライバーが**OpenClawなどのサードパーティハーネス**でのサブスクリプション枠を使用できなくなると発表した。今後は別途の従量課金オプションで対応する必要がある。Claude CodeのリードであるBoris Chernyは、「サブスクリプションはこれらのサードパーティツールの利用パターン向けには設計されていない」とコメントした。

### Claude.aiの障害（2026年4月6〜7日）
Claude.aiが大規模な障害を経験し、ログイン・音声モード・通常のChatに影響が出た。

### バージョン2.1.89〜2.1.92（2026年4月第1週リリース）
4月だけで4つのバージョンをリリース。主な変更点：
- `/powerup` コマンドの追加（v2.1.90 / 4月1日）：Claude Codeの機能をアニメーションで学べるインタラクティブなレッスン
- **`/release-notes` のインタラクティブ化**：最新版表示からバージョン選択ピッカー形式へ変更
- **MCPサイズ制限の拡大**：500K トークンまで対応（大規模プロジェクトへの活用が容易に）
- **ヘッドレス時の権限 defer 機能**：`PreToolUse` フックで「defer」オプションが追加され、CI/CDパイプラインや外部承認ゲートとの連携が可能に
- **Write ツールの diff 計算速度改善**：タブや特殊文字を含む大きなファイルで約60%高速化
- `--resume` 時のプロンプトキャッシュミス問題の修正
- 自動更新時のレート制限ダイアログが無限ループする問題の修正

### セキュリティインシデント（2026年3月31日）
2026年3月31日リリースに誤って59.8MBのJavaScriptソースマップファイルが含まれており、約512,000行の難読化されていないTypeScriptコードが露出するインシデントが発生した。

### 2026年3月の主要アップデート
- `--bare` フラグによる非同期制御の改善（スクリプト呼び出し向け）
- `--channels` パーミッションリレーの追加
- Windows向けPowerShellツールのプレビュー
- VS Codeでのレート制限警告
- インタラクティブなチャート・図・可視化・モバイルアプリの生成機能
- MacアプリのComputer Use機能（APIなしでMacアプリを操作）
- 1Mトークンコンテキストウィンドウ対応（大規模プロジェクト向け）

## 活用事例・Tips

### 非エンジニアによる業務自動化
- 経理部門で請求書確認作業を4.5時間から35分に短縮（87%削減）した事例
- 毎朝7時に自動起動し必要な情報を事前に準備する11個のAI自動化ジョブを構築
- プログラミング知識ゼロでもClaude Codeを使ったレポート作成・議事録・メール処理が可能に

### エンタープライズ導入
- Gemcookが2026年2月に全社導入を決定。導入から数日で数万行のコードを生成したメンバーも登場
- TVerでは、前提知識ゼロで大規模プロジェクトに途中参画した際にClaude Codeを活用し、不具合分析・修正・PR作成のワークフローを標準化

### 新機能Tips（2026年4月）

**ultrathink キーワード**
プロンプト内で「ultrathink」と記述すると、31,999トークンを割り当てた最高レベルの深い思考モードが起動する。

**`!` プレフィックスでbash直接実行**
`!ls` のように `!` プレフィックスを付けるとモデルトークンを消費せずにシェルコマンドを実行でき、出力がコンテキストに直接注入される。

**推奨ワークフロー**
`/plan` → 実装作業 → `/context` 確認 → `/compact` → 続きの作業 → `/diff` → コミット

### ベストプラクティス

**CLAUDE.mdの管理**
- CLAUDE.mdは200行以内に収める（それ以上だとコンテキストを消費しすぎて指示の遵守率が低下）
- Claude Codeが間違いを犯して修正された際は、同じミスを防ぐルールを`lessons.md`に書かせる

**Plan Modeの活用**
- コーディングに直接飛び込む前にPlan Modeを使い、1つ目のClaudeにプランを作成させ、2つ目のClaudeに「スタッフエンジニアとして」レビューさせる

**並行開発**
- 10〜15個のClaudeセッションを同時に実行（ターミナルタブに5個、ウェブに5〜10個）
- 各セッションはそれぞれの`git worktree`を持つので変更が衝突しない

**コンテキスト管理**
- リサーチにはサブエージェントを使用
- 新しいタスクは新しいセッションで開始
- 不要な情報をコンテキストに詰め込まない

**キーバインド**
- `Esc`: コンテキストを失わずにClaudeを停止し即座にリダイレクト可能
- `Esc+Esc` または `/rewind`: Claudeが作成したすべてのチェックポイントのスクロール可能なメニューを表示

**Skillsの活用**
- SkillsはClaude.mdとは異なり、関連するタスク時のみロードされるためコンテキストを効率的に使える
